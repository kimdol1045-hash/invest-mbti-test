# 공공화장실 찾기 - 앱인토스 미니앱 기획문서

---

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| **앱 이름 (한글)** | 급할 땐 여기 |
| **앱 이름 (영문)** | NearLoo |
| **한줄 소개** | 내 주변 공공화장실을 빠르게 찾고, 청결도를 미리 확인해요 |
| **카테고리** | 생활/편의 |
| **Primary Color** | #3182F6 (토스 블루) |

### 타겟 사용자

| 사용자 그룹 | 설명 | 니즈 |
|-------------|------|------|
| **외출 빈도가 높은 직장인** | 출퇴근, 외근, 미팅 중 화장실이 급한 상황 | 가장 가까운 화장실을 빠르게 찾고 싶어요 |
| **여행자/관광객** | 국내 여행 중 낯선 지역에서의 화장실 탐색 | 깨끗한 화장실을 미리 확인하고 싶어요 |
| **장애인/어르신** | 장애인 화장실, 접근성이 보장된 시설 필요 | 장애인용 화장실이 있는 곳만 필터링하고 싶어요 |
| **자녀를 둔 부모** | 어린이용 변기/시설이 있는 화장실 탐색 | 아이가 사용할 수 있는 화장실을 찾고 싶어요 |
| **배달/택배 라이더** | 야외 업무 중 화장실 접근이 어려운 상황 | 24시간 개방된 화장실 위치를 알고 싶어요 |

### 시장 규모 및 기회

- 토스 누적 가입자 약 3,000만명, 미니앱 이용자 5,100만명 이상 (2026년 2월 기준)
- 전국 공중화장실 약 50,000개 이상의 데이터가 공공데이터포털에 등록
- 기존 화장실 찾기 앱은 독립 앱으로만 존재하며, 토스 생태계 내 화장실 서비스는 부재

## 2. 핵심 기능

### MVP 1차 (출시 최소 기능)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **내 주변 화장실 지도** | 현재 위치 기반으로 반경 내 공공화장실을 지도에 표시해요 | P0 |
| **화장실 목록 뷰** | 거리순으로 정렬된 화장실 목록을 보여줘요 (이름, 거리, 개방시간) | P0 |
| **화장실 상세 정보** | 주소, 개방시간, 남녀 변기 수, 장애인 시설 유무 등 상세 정보를 확인할 수 있어요 | P0 |
| **장애인 화장실 필터** | 장애인용 대변기/소변기가 있는 화장실만 필터링할 수 있어요 | P0 |
| **길찾기 연동** | 선택한 화장실까지 토스 내 길찾기 또는 외부 지도앱으로 연결해요 | P1 |
| **인앱 광고(IAA)** | 화장실 목록 하단, 상세 페이지에 배너/전면 광고를 노출해요 | P1 |

### 확장 2차 (출시 이후 업데이트)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **청결도 별점 리뷰** | 사용자가 방문한 화장실의 청결도를 1~5점으로 평가하고 한줄평을 남길 수 있어요 | P2 |
| **실시간 혼잡도 (크라우드소싱)** | 사용자의 방문 데이터 기반으로 시간대별 혼잡도를 예측해요 | P2 |
| **즐겨찾기** | 자주 가는 화장실을 즐겨찾기에 저장할 수 있어요 | P2 |
| **화장실 제보** | 미등록 화장실을 사용자가 직접 제보할 수 있어요 | P2 |
| **어린이용 시설 필터** | 어린이용 변기/세면대가 있는 화장실을 필터링할 수 있어요 | P3 |
| **개방시간 알림** | 즐겨찾기한 화장실의 개방/폐쇄 시간을 알려줘요 | P3 |
| **보상형 광고** | 리뷰 작성 시 보상형 광고를 통해 토스 포인트를 지급해요 | P3 |
| **주변 편의시설 연계** | 화장실 근처 편의점, 카페 등 대안 시설도 함께 안내해요 | P3 |

## 3. 화면 구성

### 3-1. 화면 목록

#### 화면 1: 홈 (지도 뷰)
- **경로**: `intoss://nearloo` (메인)
- **구성 요소**:
  - **상단 Navigation Bar**: 앱 로고 + "급할 땐 여기" 타이틀, 검색 아이콘
  - **지도 영역**: 현재 위치 중심으로 공공화장실 마커 표시
    - 마커 색상: 일반(#3182F6 파란색), 장애인 시설 있음(#34C759 초록색)
    - 마커 탭 시 미니 카드(화장실명, 거리, 별점) 표시
  - **현재 위치 버튼**: 우측 하단, 탭 시 현재 위치로 지도 이동
  - **필터 칩**: 지도 상단에 "장애인 화장실", "24시간", "남녀공용" 필터 버튼
  - **하단 시트 (Bottom Sheet)**: 위로 스와이프하면 목록 뷰로 전환
  - **배너 광고**: 하단 시트 내부 목록 사이에 네이티브 광고 배치

#### 화면 2: 목록 뷰
- **경로**: 홈 화면 Bottom Sheet 확장
- **구성 요소**:
  - **정렬 옵션**: 거리순(기본) / 별점순 (2차)
  - **화장실 카드 리스트**: TDS ListRow 컴포넌트 사용
    - 화장실명, 소재지 주소, 현재 위치 기준 거리, 개방시간
    - 장애인 시설 아이콘 배지, 청결도 별점 (2차)
  - **배너 광고**: 목록 5개마다 네이티브 배너 광고 1개 삽입
  - **무한 스크롤**: 추가 데이터 로드

#### 화면 3: 화장실 상세
- **경로**: `intoss://nearloo/detail/:id`
- **구성 요소**:
  - **상단 Navigation Bar**: 뒤로가기 버튼, 공유 버튼
  - **화장실명 헤더**: 이름, 관리기관명, 전화번호 (탭 시 전화 연결)
  - **위치 미니맵**: 해당 화장실 위치를 작은 지도로 표시
  - **상세 정보 카드**:
    - 소재지 주소 (도로명 + 지번)
    - 개방시간
    - 설치연도
  - **시설 현황 테이블**:
    - 남성용: 대변기 수, 소변기 수, 장애인용 대변기 수, 장애인용 소변기 수, 어린이용 대변기 수, 어린이용 소변기 수
    - 여성용: 대변기 수, 장애인용 대변기 수, 어린이용 대변기 수
    - 남녀공용 여부
  - **길찾기 버튼**: TDS BottomCTA 컴포넌트, 탭 시 지도앱 연동
  - **청결도 리뷰 섹션 (2차)**: 평균 별점, 리뷰 목록
  - **전면 광고**: 상세 페이지 진입 3회마다 1회 전면 광고 노출

#### 화면 4: 검색
- **경로**: `intoss://nearloo/search`
- **구성 요소**:
  - **검색 바**: 장소명, 주소 키워드 검색
  - **최근 검색어**: 최근 검색한 키워드 칩
  - **검색 결과**: ListRow 형태로 결과 표시
  - **검색 없을 때**: "검색 결과가 없어요" 안내 + 주변 화장실 추천

#### 화면 5: 리뷰 작성 (2차)
- **경로**: `intoss://nearloo/review/:id`
- **구성 요소**:
  - **별점 입력**: 1~5점 별점 선택
  - **한줄평 입력**: 최대 100자 텍스트 입력
  - **제출 버튼**: TDS BottomCTA 컴포넌트
  - **보상형 광고 (선택)**: 리뷰 제출 후 보상형 광고 시청 시 토스 포인트 지급

### 3-2. 화면 플로우

```
[토스 앱 진입]
     │
     ▼
[홈 - 지도 뷰] ──── 검색 아이콘 탭 ────→ [검색 화면]
     │                                        │
     │ 마커 탭                                 │ 결과 탭
     ▼                                        ▼
[마커 미니 카드] ──── 카드 탭 ────→ [화장실 상세]
     │                                  │
     │ 스와이프 업                       │ 길찾기 탭
     ▼                                  ▼
[목록 뷰 (Bottom Sheet)] ────→ [외부 지도앱 / 토스 길찾기]
     │
     │ 리스트 아이템 탭
     ▼
[화장실 상세] ──── 리뷰 작성 탭 (2차) ────→ [리뷰 작성]
```

### 3-3. TDS 컴포넌트 매핑

| 화면 요소 | TDS 컴포넌트 | 비고 |
|-----------|-------------|------|
| 상단 바 | `Navigation` | 타이틀 + 액션 버튼 |
| 화장실 목록 | `ListRow` | 제목, 부제, 우측 거리 표시 |
| 필터 버튼 | `Chip` | 토글형 필터 칩 |
| 길찾기 버튼 | `BottomCTA` | 하단 고정 CTA 버튼 |
| 별점 | `Rating` (커스텀) | 별 아이콘 1~5점 |
| 상세 정보 | `Card` / `ListRow` | 정보 그룹핑 |
| 검색 바 | `TextField` | 검색 입력 필드 |
| 빈 상태 | `EmptyState` | 결과 없음 안내 |
| 토스트 알림 | `Toast` | 성공/에러 피드백 |

### 4-3. 데이터 저장

| 구분 | 저장소 | 용도 |
|------|--------|------|
| **화장실 데이터 캐시** | 앱인토스 네이티브 스토리지 (`get`/`set`) | API 응답 로컬 캐싱 (24시간 TTL) |
| **즐겨찾기 (2차)** | 앱인토스 네이티브 스토리지 | 사용자별 즐겨찾기 목록 |
| **최근 검색어** | 앱인토스 네이티브 스토리지 | 최근 검색 키워드 (최대 10개) |
| **리뷰 데이터 (2차)** | 자체 서버 DB (PostgreSQL) + mTLS | 사용자 리뷰/별점 저장 |
| **사용자 설정** | 앱인토스 네이티브 스토리지 | 필터 설정, 지도 줌 레벨 등 |

### 4-4. 서버 구성 (2차 이후)

MVP에서는 서버 없이 클라이언트에서 공공데이터 API를 직접 호출하는 구조로 시작하고, 리뷰 기능 추가 시 서버를 구축해요.

```
[MVP 구조]
토스 앱 → 미니앱(WebView) → 공공데이터포털 API
                           → Kakao Maps API

[2차 확장 구조]
토스 앱 → 미니앱(WebView) → 자체 서버 (mTLS) → 공공데이터포털 API
                           → Kakao Maps API     → PostgreSQL (리뷰 DB)
```

## 5. 수익화 전략

### 5-1. 인앱 광고 (IAA) - 핵심 수익원

앱인토스 In-app Ads 2.0을 활용하며, 토스 Ads와 Google AdMob을 모두 지원해요.

| 광고 유형 | 배치 위치 | 노출 조건 | 예상 효과 |
|-----------|-----------|-----------|-----------|
| **배너 광고** | 목록 뷰 내 5개 항목마다 1개 | 항상 노출 | 안정적 기본 수익 |
| **전면 광고** | 화장실 상세 페이지 진입 | 3회 진입마다 1회 | 높은 CPM 확보 |
| **보상형 광고 (2차)** | 리뷰 작성 완료 후 | 사용자 선택 시청 | 리뷰 참여율 + 광고 수익 동시 확보 |

### 5-2. 수익 예상 시나리오

| 지표 | 보수적 | 중간 | 낙관적 |
|------|--------|------|--------|
| DAU | 5,000 | 15,000 | 50,000 |
| 1인당 일 페이지뷰 | 3 | 5 | 8 |
| 배너 CPM (원) | 500 | 800 | 1,200 |
| 전면 광고 CPM (원) | 3,000 | 5,000 | 8,000 |
| **월 예상 수익 (원)** | **약 45만** | **약 360만** | **약 2,880만** |

> **참고**: 앱인토스 인앱 광고 수수료는 CBT 기간 동안 15%이며, 현재(2026년 3월 기준) 0%로 운영 중이에요.

### 5-3. 수익화 로드맵

```
MVP 출시 (1개월차)
  └── 배너 광고 + 전면 광고 → 기본 수익 확보

2차 업데이트 (3개월차)
  └── 보상형 광고 (리뷰 작성 보상) → 참여 + 수익 시너지

3차 확장 (6개월차)
  └── 토스 포인트 프로모션 연계 → MAU 확대 → 광고 단가 상승
```

## 8. 경쟁력 & 차별점

### 8-1. 기존 서비스 비교

| 비교 항목 | 급할 땐 여기 (NearLoo) | 기존 화장실 찾기 앱 (독립 앱) | 네이버/카카오 지도 |
|-----------|----------------------|------------------------------|-------------------|
| **접근성** | 토스 앱 내 즉시 이용 (설치 불필요) | 별도 앱 설치 필요 | 화장실 전용 검색 기능 부재 |
| **사용자 풀** | 토스 3,000만 가입자 즉시 접근 | 자체 마케팅으로 사용자 확보 | 일반 장소 검색에 묻힘 |
| **화장실 특화 정보** | 변기 수, 장애인 시설, 개방시간 등 상세 | 기본 위치 정보 위주 | 화장실 세부 정보 없음 |
| **장애인 접근성** | 장애인 화장실 전용 필터 | 일부 앱만 지원 | 미지원 |
| **수익화** | 토스 IAA 생태계 활용 | 자체 광고 네트워크 구축 필요 | N/A |
| **개발/운영 비용** | 서버리스 MVP 가능, 공공데이터 무료 | 서버 인프라 필요 | N/A |

### 8-2. 핵심 차별점

1. **토스 생태계 레버리지**: 3,000만 토스 사용자에게 별도 설치 없이 즉시 노출돼요. 앱 다운로드 허들이 전혀 없어요.

2. **공공데이터 기반 신뢰성**: 행정안전부가 관리하는 공식 공공데이터를 사용하므로 정보의 정확성과 신뢰성이 높아요. 데이터 수집 비용이 0원이에요.

3. **장애인/어린이 접근성 특화**: 기존 서비스에서 소홀했던 장애인 화장실, 어린이용 시설 정보를 전면에 배치해요. 사회적 가치와 서비스 차별점을 동시에 확보할 수 있어요.

4. **서버리스 MVP**: 공공데이터 API를 클라이언트에서 직접 호출하는 구조로, 서버 운영 비용 없이 MVP를 출시할 수 있어요. 초기 비용이 거의 들지 않아요.

5. **크라우드소싱 리뷰 확장성 (2차)**: 토스 사용자 기반의 청결도 리뷰 시스템으로 데이터 품질을 지속적으로 개선할 수 있어요. 보상형 광고와 연계하면 리뷰 참여율도 높일 수 있어요.

6. **즉시 수익화**: 출시 첫날부터 인앱 광고(IAA)로 수익이 발생해요. 공공데이터가 무료이고 서버 비용이 없으므로 손익분기점이 매우 낮아요.

### 8-3. 리스크 및 대응

| 리스크 | 영향도 | 대응 방안 |
|--------|--------|-----------|
| 공공데이터 품질 이슈 (누락/오류) | 중 | 지자체별 보완 데이터 병행 활용, 사용자 제보 기능 (2차) |
| 공공데이터 API 호출 제한 | 중 | 클라이언트 캐싱 (24시간 TTL), API 호출 횟수 확대 신청 |
| 유사 서비스 출현 | 하 | 리뷰/청결도 데이터로 선점 효과 확보, 토스 생태계 내 첫 진입자 이점 |
| 토스 정책 변경 | 중 | 정책 업데이트 모니터링, 유연한 광고 배치 구조 |
| 위치 권한 거부 사용자 | 중 | 주소 검색 기반 대체 UX 제공, 위치 권한 필요성 안내 |
