# 야간/휴일 약국 찾기 - 앱인토스 미니앱 기획문서

---

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| **앱 이름 (한글)** | 밤약국 |
| **앱 이름 (영문)** | NightPharm |
| **한줄 소개** | 지금 열려 있는 가장 가까운 약국을 바로 찾아드려요 |
| **카테고리** | 생활/건강 |
| **Primary Color** | `#3182F6` (토스 블루) |

### 타겟 사용자
- **1차 타겟**: 야간/주말/공휴일에 급하게 약이 필요한 20~50대 성인
- **2차 타겟**: 영유아 자녀를 둔 부모 (야간 응급 상황)
- **3차 타겟**: 고령 부모를 돌보는 보호자, 1인 가구
- **핵심 사용 시나리오**: "밤 11시, 아이가 갑자기 열이 나는데 지금 문 연 약국이 어디지?"

### 기획 배경
- 전국 약국 약 24,000개 중 야간/공휴일 운영 약국은 극소수
- 기존 서비스(e약은요, 약국찾기 앱)는 실시간 운영 여부 확인이 어렵고 UX가 불편함
- 토스 사용자 3,000만명 중 야간 의료 서비스 수요가 높은 20~40대가 핵심 이용층

## 2. 핵심 기능

### MVP 1차 (2주 내 개발)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **현재 위치 기반 약국 검색** | GPS 기반으로 현재 열려 있는 주변 약국 목록 표시 | P0 |
| **지도 뷰** | 약국 위치를 지도에 핀으로 표시, 현재 위치 중심 | P0 |
| **약국 상세 정보** | 약국명, 주소, 전화번호, 요일별 영업시간, 거리, 길찾기 | P0 |
| **야간/공휴일 필터** | "지금 영업 중", "야간 운영", "공휴일 운영" 필터 토글 | P0 |
| **전화 연결** | 약국 전화번호 원터치 발신 | P1 |
| **지역 검색** | 시/구/동 단위 주소 검색으로 약국 찾기 | P1 |
| **배너 광고 (AdMob)** | 리스트 하단 배너 광고 노출 | P1 |

### 확장 2차 (이후 업데이트)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **즐겨찾기 약국** | 자주 이용하는 약국 저장 (네이티브 스토리지) | P2 |

| **약국 영업 정보 제보** | 사용자가 실제 영업 여부를 제보하는 크라우드소싱 | P3 |
| **의약품 재고 확인** | 특정 의약품 보유 약국 검색 (공공API 연동) | P3 |
| **길찾기 연동** | 토스 내 지도 또는 외부 내비 앱 연동 | P3 |
| **Smart Message** | 주말/공휴일 전날 저녁, 야간 약국 안내 메시지 발송 | P3 |

## 3. 화면 구성

### 3-1. 화면 목록

#### (1) 홈 화면 (메인)
- **상단**: `Navigation` 컴포넌트 - "밤약국" 타이틀, 검색 아이콘
- **현재 위치 표시**: 현재 위치 주소 텍스트 + 위치 재설정 버튼
- **필터 영역**: `Tab` 또는 `SegmentedControl` - "지금 영업 중" | "야간 약국" | "공휴일 약국"
- **약국 리스트**: `ListRow` 컴포넌트 반복
  - 왼쪽: 약국 아이콘 (영업중: Blue, 마감: Grey)
  - 타이틀: 약국명
  - 서브타이틀: 주소 + 거리 (예: "500m")
  - 우측: 영업 상태 `Badge` ("영업 중" Green / "마감" Grey / "곧 마감" Red)
- **하단**: AdMob 배너 광고 영역
- **FAB(플로팅 버튼)**: 지도 보기 전환 버튼

#### (2) 지도 화면
- **상단**: `Navigation` 컴포넌트 - 뒤로가기, "주변 약국 지도"
- **지도 영역**: 전체 화면 지도 (현재 위치 중심)
  - 약국 위치 마커 (영업중: Blue 핀, 마감: Grey 핀)
  - 마커 탭 시 하단 미니 카드 표시
- **하단 미니 카드**: 약국명, 영업 상태, 거리, "상세보기" 버튼
- **필터 칩**: 지도 상단에 "야간" / "공휴일" 필터 칩

#### (3) 약국 상세 화면
- **상단**: `Navigation` 컴포넌트 - 뒤로가기, 약국명
- **기본 정보 섹션**: `ListRow` 구성
  - 약국명 (큰 텍스트)
  - 영업 상태 `Badge`
  - 주소 (복사 버튼)
  - 전화번호 (탭하면 전화 연결)
- **영업시간 섹션**: `ListHeader` + `ListRow` 반복
  - 요일별 영업시간 목록 (월~일, 공휴일)
  - 현재 요일 하이라이트 처리
- **위치 섹션**: 소형 지도 + "길찾기" 버튼
- **하단 CTA**: `BottomCTA` - "전화하기" 버튼 (Primary Blue)

#### (4) 지역 검색 화면
- **상단**: `Navigation` 컴포넌트 - 뒤로가기, "지역 검색"
- **검색 필드**: `SearchField` 컴포넌트 - 시/구/동 검색
- **최근 검색 목록**: `ListRow` 반복
- **검색 결과**: 주소 목록 `ListRow` - 탭 시 해당 지역 약국 리스트로 이동

#### (5) 즐겨찾기 화면 (2차)
- **상단**: `Navigation` 컴포넌트 - 뒤로가기, "즐겨찾기"
- **즐겨찾기 약국 리스트**: `ListRow` 반복 (홈 화면과 동일 구조)
- **빈 상태**: 즐겨찾기 없을 시 안내 일러스트 + "주변 약국 찾아보기" 버튼

### 3-2. 화면 플로우

```
[토스 앱]
    |
    v
[홈 화면 - 약국 리스트] ──── [지도 화면]
    |         |                    |
    |         v                    v
    |    [지역 검색] ──> [검색 결과 리스트]
    |                          |
    v                          v
[약국 상세 화면] <──────────────┘
    |
    ├── 전화하기 (네이티브 전화 연결)
```

### 4-2. 외부 API (공공데이터포털)

#### (1) 국립중앙의료원 - 전국 약국 정보 조회 서비스 (핵심 API)

| 항목 | 내용 |
|------|------|
| **데이터 ID** | 15000576 |
| **제공기관** | 국립중앙의료원 |
| **엔드포인트** | `http://apis.data.go.kr/B552657/ErmctInsttInfoInqireService/getParmacyListInfoInqire` |
| **주요 기능** | 주소별, 진료요일별, 시간대별 약국 조회 |
| **응답 형식** | XML / JSON |
| **트래픽** | 개발 계정 1,000,000건 / 운영 계정 확대 가능 |
| **비용** | 무료 |

**주요 요청 파라미터:**

| 파라미터 | 설명 | 예시 |
|----------|------|------|
| `serviceKey` | 공공데이터포털 인증키 | (발급 필요) |
| `Q0` | 시도 코드/이름 | "서울특별시" |
| `Q1` | 시군구 코드/이름 | "강남구" |
| `QT` | 진료 시간 (HHMM) | "2300" |
| `QN` | 약국 이름 | "OO약국" |
| `ORD` | 정렬 기준 | "NAME" |
| `pageNo` | 페이지 번호 | 1 |
| `numOfRows` | 한 페이지 결과 수 | 20 |

**주요 응답 필드:**

| 필드 | 설명 |
|------|------|
| `dutyName` | 약국 이름 |
| `dutyAddr` | 약국 주소 |
| `dutyTel1` | 전화번호 |
| `wgs84Lat` | 위도 (WGS84) |
| `wgs84Lon` | 경도 (WGS84) |
| `dutyTime1s` ~ `dutyTime8s` | 요일별 영업 시작시간 (1=월 ~ 7=일, 8=공휴일) |
| `dutyTime1c` ~ `dutyTime8c` | 요일별 영업 종료시간 |
| `hpid` | 기관 고유 ID |

> 참고: 데이터는 약국에서 제공하는 정보 기반이므로, 실제 영업 여부는 전화 확인을 안내해야 해요.

#### (2) 건강보험심사평가원 - 약국정보서비스 (보조 API)

| 항목 | 내용 |

| **제공기관** | 건강보험심사평가원 |

| **트래픽** | 개발 계정 10,000건 / 운영 계정 확대 가능 |

#### (3) 전국약국표준데이터 (보조 데이터)

| 항목 | 내용 |
|------|------|

### 4-3. 데이터 저장

| 저장소 | 용도 | 기술 |
|--------|------|------|
| **네이티브 스토리지** | 즐겨찾기, 최근 검색, 설정 | 앱인토스 `get/set/clear` API |
| **클라이언트 캐시** | 약국 데이터 캐싱 (TTL 1시간) | localStorage / sessionStorage |
| **별도 서버 불요** | MVP는 서버 없이 공공API + 클라이언트만으로 구현 | - |

> MVP 단계에서는 별도 백엔드 서버 없이, 클라이언트에서 공공데이터 API를 직접 호출하는 구조로 개발해요. 사용자 데이터(즐겨찾기 등)는 앱인토스 네이티브 스토리지에 저장해요.

앱인토스 SDK v1.11.0+에서 지원하는 In-app Ads 2.0 (토스 Ads + Google AdMob) 활용.

| 광고 유형 | 위치 | 노출 시점 | 예상 CPM | 단계 |
|-----------|------|-----------|----------|------|
| **배너 광고** | 약국 리스트 하단 | 상시 노출 | $0.5~2.0 | MVP 1차 |

### 5-2. 수익 예상 (보수적 추정)

| 지표 | MVP 1개월 | 안정화 3개월 후 |

| 월 예상 수익 | 약 5~15만원 | 약 30~90만원 |

### 5-3. 수수료 구조

| 항목 | 비율 | 비고 |
|------|------|------|

- 광고 영역 명확 구분: 콘텐츠와 광고 영역이 혼동되지 않도록 처리

### 6-2. 조건부 허용 카테고리 검토

| 카테고리 | 해당 여부 | 비고 |
|----------|-----------|------|
| 의료 서비스 | **부분 해당** | 약국 정보 제공은 "비치료" 영역이므로 허용. 단, 의료 행위(처방, 진단 등)는 포함하지 않음 |
| 쇼핑 | 해당 없음 | 상품 판매 기능 없음 |

### 6-3. 위반 가능 항목 및 대응

| 잠재적 위반 | 위험도 | 대응 방안 |
|-------------|--------|-----------|
| **의료 정보 제공 관련** | 낮음 | 약국 위치/영업시간만 제공, 의약품 추천이나 의료 조언 기능 포함하지 않음 |
| **개인정보 (위치)** | 중간 | 위치 권한 사용 목적 명시, 개인정보 처리방침 작성 필수 |

## 8. 경쟁력 & 차별점

### 8-1. 기존 서비스 비교

| 서비스 | 장점 | 단점 |
|--------|------|------|
| **e약은요** (식약처 공식) | 공식 데이터, 의약품 정보 | UX 구식, 로딩 느림, 야간 필터 없음 |
| **굿닥** | 병원+약국 통합 | 약국 기능이 부가적, 무거운 앱 |
| **약국찾기 앱** (개별) | 전용 기능 | 별도 설치 필요, 사용자 적음 |
| **네이버 지도** | 사용자 많음, UX 우수 | 야간/공휴일 전용 필터 없음, 약국 영업시간 부정확 |
| **카카오맵** | 사용자 많음 | 당번약국 정보 미제공 |

### 8-2. 밤약국의 차별점

| 차별점 | 설명 |
|--------|------|
| **토스 네이티브 접근성** | 별도 앱 설치 없이 토스 앱 내에서 바로 사용. 3,000만 토스 사용자에게 즉시 노출 가능 |
| **야간/휴일 특화 UX** | "지금 열려 있는 약국"에 집중한 단일 목적 서비스. 불필요한 기능 없이 빠른 검색 |
| **원터치 전화 연결** | 약국 정보 확인 후 1탭으로 전화 연결. 급한 상황에서의 빠른 행동 유도 |
| **토스 디자인 시스템** | TDS 기반의 익숙하고 깔끔한 UI. 토스 사용자에게 자연스러운 경험 |
| **서버리스 경량 구조** | 공공 API + 클라이언트만으로 운영. 서버 비용 0원, 유지보수 최소화 |
| **실시간 영업 상태 판별** | 현재 시각 기준으로 공공데이터의 요일별 영업시간을 자동 계산하여 영업 상태 실시간 표시 |
| **Smart Message 활용** | 주말/공휴일 전날 저녁, 토스 알림으로 "내 주변 야간 약국" 안내 가능 (2차) |

### 8-3. 핵심 경쟁 우위 요약

1. **제로 설치 비용**: 토스 앱 내 미니앱이므로 별도 설치 불필요 → 진입 장벽 최소화
2. **3,000만 잠재 사용자**: 토스 사용자 기반 즉시 접근 가능
3. **단일 목적 집중**: "야간/휴일에 열린 약국 찾기" 하나에만 집중한 최적화된 UX
4. **무료 공공 API**: 서버/데이터 비용 0원으로 MVP 운영 가능
5. **빠른 MVP**: 2주 내 개발 완료 → 시장 검증 후 확장 가능
