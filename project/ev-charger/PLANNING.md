# 전기차 충전소 찾기 - 앱인토스 미니앱 기획문서

> 작성일: 2026-03-06
> 프로젝트 유형: 앱인토스 WebView 미니앱 (비게임)
> 수익 모델: IAA (인앱광고)

---

## 1. 앱 개요

### 앱 이름
- **한글**: 충전어디
- **영문**: ChargeFinder

### 한줄 소개
내 주변 전기차 충전소를 실시간으로 찾고, 사용 가능 여부를 한눈에 확인하는 서비스예요.

### 타겟 사용자
| 구분 | 설명 |
|------|------|
| **주요 타겟** | 전기차 소유자 (20~50대, 테슬라/현대/기아 EV 오너) |
| **보조 타겟** | 전기차 구매 예정자, 렌터카/카셰어링으로 전기차를 이용하는 사용자 |
| **사용 시나리오** | 외출 중 급하게 충전소를 찾을 때, 장거리 운전 전 경로 상의 충전소를 미리 확인할 때, 자주 가는 충전소의 실시간 상태를 확인할 때 |

### 핵심 가치
- **실시간성**: 환경부 API를 통해 충전기 상태(충전대기/충전중/고장 등)를 실시간으로 제공해요
- **편의성**: 토스 앱 내에서 별도 설치 없이 바로 사용할 수 있어요
- **간결함**: 복잡한 기능 없이 "가장 가까운 사용 가능한 충전소"를 빠르게 찾을 수 있어요

## 2. 핵심 기능

### MVP 1차 (필수 기능)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **주변 충전소 지도** | 현재 위치 기반으로 주변 충전소를 지도 위에 마커로 표시해요. 충전기 상태에 따라 마커 색상이 달라져요 (초록: 사용 가능, 빨강: 사용 불가, 회색: 통신이상) | P0 |
| **충전소 목록 보기** | 거리순으로 정렬된 충전소 리스트를 제공해요. 충전소명, 주소, 거리, 충전기 타입, 사용 가능 대수를 표시해요 | P0 |
| **충전소 상세 정보** | 충전소를 선택하면 상세 정보(운영 시간, 충전기 타입별 현황, 주차 요금, 운영 기관, 연락처 등)를 확인할 수 있어요 | P0 |
| **충전기 타입 필터** | DC콤보, DC차데모, AC완속, AC3상 등 내 차에 맞는 충전기 타입으로 필터링할 수 있어요 | P0 |
| **실시간 상태 확인** | 충전대기, 충전중, 운휴, 점검, 통신이상 등 충전기 상태를 실시간으로 확인할 수 있어요 | P0 |
| **길찾기 연동** | 충전소 상세에서 "길찾기" 버튼을 누르면 토스 내 지도 또는 외부 네비게이션 앱으로 연결해요 | P1 |
| **주소 검색** | 특정 지역의 충전소를 검색할 수 있어요 | P1 |

### 확장 2차 (추가 기능)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **즐겨찾기** | 자주 이용하는 충전소를 즐겨찾기에 저장할 수 있어요 | P2 |
| **충전소 알림** | 즐겨찾기한 충전소의 충전기가 사용 가능해지면 알림을 받을 수 있어요 | P2 |
| **충전 요금 비교** | 운영 사업자별 kWh당 충전 요금을 비교할 수 있어요 | P2 |
| **혼잡도 통계** | 시간대별 충전소 혼잡도를 그래프로 확인할 수 있어요 (과거 데이터 기반) | P3 |
| **경로 충전 플래너** | 출발지-목적지를 입력하면 경로 상의 충전소를 추천해줘요 | P3 |
| **사용자 리뷰** | 충전소에 대한 사용자 리뷰와 편의시설 정보를 공유할 수 있어요 | P3 |

## 3. 화면 구성

### 화면 목록

#### 3-1. 홈 화면 (지도 뷰)
- **구성 요소**:
  - TDS `Navigation` 상단바 (앱 제목 "충전어디", 검색 아이콘, 리스트 전환 아이콘)
  - 지도 영역 (화면의 70% 차지)
  - 충전소 마커 (상태별 색상 구분)
    - 초록(`#34C759`): 사용 가능 충전기 있음
    - 빨강(`#FF3B30`): 전체 사용 불가/충전 중
    - 회색(`#8B95A1`): 통신이상/점검 중
  - 하단 충전기 타입 필터 칩 (가로 스크롤, TDS `Chip` 컴포넌트)
  - "내 위치로" 플로팅 버튼
  - 지도 위 마커 탭 시 간략 정보 바텀시트 노출
- **상호작용**:
  - 지도 이동/줌 시 해당 영역의 충전소 자동 로드
  - 마커 탭 -> 간략 정보 바텀시트 -> 탭 시 상세 화면 이동
  - 필터 칩 선택 시 해당 타입 충전소만 표시

#### 3-2. 목록 화면 (리스트 뷰)
- **구성 요소**:
  - TDS `Navigation` 상단바 (검색 아이콘, 지도 전환 아이콘)
  - 정렬 옵션 (거리순/사용가능순)
  - TDS `ListRow` 기반 충전소 리스트
    - 충전소명 (좌측 상단, 굵은 텍스트)
    - 주소 (충전소명 하단, 서브 텍스트)
    - 거리 (우측 상단, 토스 블루 `#3182F6`)
    - 충전기 상태 뱃지 (사용가능 N대 / 전체 M대)
    - 충전기 타입 아이콘
  - 리스트 하단 인앱광고 배너 영역
- **상호작용**:
  - 무한 스크롤 (페이지네이션)
  - 리스트 아이템 탭 -> 충전소 상세 화면 이동
  - 당겨서 새로고침 (Pull-to-refresh)

#### 3-3. 충전소 상세 화면
- **구성 요소**:
  - TDS `Navigation` 상단바 (뒤로가기, 충전소명, 즐겨찾기 아이콘)
  - 미니 지도 (충전소 위치 표시, 탭 시 전체 지도 뷰로 확대)
  - 기본 정보 섹션
    - 충전소명, 주소 (복사 버튼 포함)
    - 운영 시간
    - 운영 기관 / 연락처 (탭 시 전화 연결)
    - 주차 요금 (무료/유료)
  - 충전기 현황 섹션 (TDS `ListRow` 활용)
    - 충전기별 타입, 충전 용량(kW), 현재 상태
    - 상태별 색상 뱃지 (충전대기: 초록, 충전중: 주황, 운휴/점검: 빨강, 통신이상: 회색)
    - 마지막 상태 갱신 시각
  - TDS `BottomCTA` - "길찾기" 버튼 (토스 블루 `#3182F6`)
- **상호작용**:
  - 길찾기 버튼 -> 네비게이션 앱 선택 액션시트
  - 연락처 탭 -> 전화 연결
  - 주소 복사 버튼 -> 클립보드 복사 + 토스트 알림

#### 3-4. 검색 화면
- **구성 요소**:
  - TDS 검색 입력 필드 (자동 포커스)
  - 최근 검색어 (로컬 저장)
  - 검색 결과 리스트 (TDS `ListRow`)
  - 검색 결과 없음 안내 메시지
- **상호작용**:
  - 입력 시 자동완성 (주소/충전소명 기반)
  - 검색 결과 항목 탭 -> 충전소 상세 화면 이동

#### 3-5. 즐겨찾기 화면 (2차)
- **구성 요소**:
  - TDS `Navigation` 상단바 ("즐겨찾기")
  - 즐겨찾기한 충전소 리스트 (TDS `ListRow`)
  - 각 충전소의 실시간 상태 요약
  - 비어 있을 때 안내 메시지 ("자주 가는 충전소를 추가해보세요")
- **상호작용**:
  - 스와이프로 즐겨찾기 삭제
  - 항목 탭 -> 충전소 상세 화면 이동

### 화면 플로우

```
[앱 진입 (토스 앱)]
       |
       v
[홈 화면 - 지도 뷰] <---> [목록 화면 - 리스트 뷰]
       |                          |
       |--- [검색 화면] <----------+
       |
       v (마커 탭 또는 리스트 항목 탭)
[충전소 상세 화면]
       |
       |--- [길찾기] -> 외부 네비게이션 앱
       |--- [전화 연결] -> 전화 앱
       |--- [즐겨찾기 추가/삭제]
       |
       v (2차)
[즐겨찾기 화면]
```

```
상세 화면 플로우:

[홈 지도]
  |-- 마커 탭 -> [바텀시트: 간략 정보]
  |                |-- 탭 -> [충전소 상세]
  |                               |-- 길찾기 버튼 -> [네비 앱 선택 시트]
  |                               |-- 전화 아이콘 -> [전화 앱]
  |-- 필터 칩 탭 -> [필터링된 마커 표시]
  |-- 검색 아이콘 -> [검색 화면]
  |                    |-- 결과 항목 탭 -> [해당 충전소 위치로 지도 이동]
  |-- 리스트 아이콘 -> [목록 화면]
                         |-- 항목 탭 -> [충전소 상세]
```

## 4. 기술 스택

### 프론트엔드

| 항목 | 기술 | 설명 |
|------|------|------|
| **프레임워크** | `@apps-in-toss/web-framework` (React) | 앱인토스 WebView 미니앱 공식 프레임워크 |
| **디자인 시스템** | `@toss/tds-mobile` | TDS(Toss Design System) 모바일 컴포넌트 필수 사용 |

| **상태 관리** | React Query (TanStack Query) | API 데이터 캐싱 및 실시간 갱신 관리 |

| **설정** | `granite.config.ts` | 앱인토스 프로젝트 설정 파일 |
| **빌드** | 앱인토스 CLI | 앱인토스 표준 빌드 도구 |

### 외부 API

#### 한국환경공단 전기자동차 충전소 정보 API (주요 데이터 소스)

- **제공처**: 공공데이터포털 (data.go.kr)
- **API 링크**: [한국환경공단_전기자동차 충전소 정보](https://www.data.go.kr/data/15076352/openapi.do)
- **인증**: 공공데이터포털 API 키 (ServiceKey)
- **데이터 포맷**: XML / JSON
- **호출 제한**: 개발 계정 1,000건/일, 운영 계정 신청 시 확대 가능
- **승인 방식**: 자동 승인 (신청 즉시 사용 가능)

| 엔드포인트 | URL | 용도 |
|-----------|-----|------|
| **충전소 정보 조회** | `http://apis.data.go.kr/B552584/EvCharger/getChargerInfo` | 충전소 기본 정보 (위치, 타입, 운영 정보 등) |
| **충전기 상태 조회** | `http://apis.data.go.kr/B552584/EvCharger/getChargerStatus` | 충전기 실시간 상태 (충전대기/충전중/고장 등) |

**주요 요청 파라미터:**

| 파라미터 | 설명 | 필수 |
|----------|------|------|
| `serviceKey` | 공공데이터포털 인증키 | Y |
| `pageNo` | 페이지 번호 | N |
| `numOfRows` | 한 페이지 결과 수 | N |
| `zscode` | 지역코드 (시/군/구 단위 필터링) | N |

**주요 응답 필드:**

| 필드명 | 설명 | 예시 |
|--------|------|------|
| `statNm` | 충전소명 | "서울역 급속충전소" |
| `statId` | 충전소 ID | "ME000001" |
| `chgerId` | 충전기 ID | "01" |
| `chgerType` | 충전기 타입 | 01: DC차데모, 02: AC완속, 03: DC차데모+AC3상, 04: DC콤보, 05: DC차데모+DC콤보, 06: DC차데모+AC3상+DC콤보, 07: AC3상 |
| `addr` | 주소 | "서울특별시 용산구..." |
| `lat` | 위도 | 37.5547 |
| `lng` | 경도 | 126.9707 |
| `useTime` | 이용가능시간 | "24시간 이용가능" |
| `busiNm` | 운영기관명 | "한국전력" |
| `busiCall` | 운영기관 연락처 | "1661-9408" |
| `stat` | 충전기 상태 | 1: 통신이상, 2: 충전대기, 3: 충전중, 4: 운휴, 5: 점검중, 9: 상태미확인 |
| `statUpdDt` | 상태 갱신일시 | "20260306143000" |
| `lastTsdt` | 마지막 충전 시작일시 | "20260306120000" |
| `lastTedt` | 마지막 충전 종료일시 | "20260306133000" |
| `nowTsdt` | 현재 충전 시작일시 | "20260306140000" |
| `output` | 충전용량 (kW) | "50" |
| `method` | 충전방식 | "단독", "동시" |
| `parkingFree` | 주차료 무료 여부 | "Y" / "N" |
| `limitYn` | 이용자 제한 여부 | "Y" / "N" |
| `limitDetail` | 이용 제한 사유 | "아파트 입주민 전용" |
| `zcode` | 시도 코드 | "11" (서울) |

### 데이터 저장

| 저장소 | 용도 | 기술 |
|--------|------|------|
| **LocalStorage** | 즐겨찾기 목록, 최근 검색어, 필터 설정, 내 차 충전기 타입 | 브라우저 내장 저장소 |
| **React Query Cache** | API 응답 캐싱 (5분 TTL) | 인메모리 캐시 |
| **서버 DB (2차)** | 사용자 리뷰, 혼잡도 통계 데이터 | 앱인토스 서버 + mTLS 인증 |

> **참고**: MVP 단계에서는 별도 백엔드 서버 없이 프론트엔드에서 환경부 API를 직접 호출하는 구조로 시작해요. 2차 확장 시 자체 백엔드 서버를 구축하고, mTLS 인증서 기반 통신을 적용해요.

## 5. 수익화 전략

### IAA (인앱광고) - 앱인토스 인앱광고 시스템

앱인토스에서 제공하는 인앱광고(IAA) 시스템을 활용해요. 출시 첫날부터 바로 광고 수익이 발생하며, 구글 애드몹 광고 정책을 따르고 있어요.

#### 광고 유형 및 배치 전략

| 광고 유형 | 배치 위치 | 노출 시점 | 예상 eCPM |
|-----------|-----------|-----------|-----------|
| **전면형 (Interstitial)** | - | 충전소 상세 정보 진입 시 (5회 조회마다 1회) | 높음 |
| **보상형 (Rewarded)** | 즐겨찾기 화면 | "광고 보고 즐겨찾기 슬롯 추가하기" (2차) | 매우 높음 |

#### 광고 정책 준수사항
- 앱인토스 인앱광고는 구글 애드몹 광고 정책을 따라요
- 전면형 광고는 사용자 경험을 해치지 않는 자연스러운 전환 시점에만 노출해요
- 충전소를 급하게 찾는 메인 플로우(지도/목록)에는 광고를 넣지 않아요
- 닫기 버튼을 명확히 제공하고, 기만적인 광고 노출(다크패턴)을 하지 않아요

#### 연동 절차
1. 앱인토스 콘솔에서 약관 동의
2. 정산 정보 입력 (영업일 2~3일 검토)
3. 광고 그룹 생성
4. 광고 SDK 코드 삽입

#### 수익 예상 (월간)

| 지표 | 보수적 | 낙관적 |
|------|--------|--------|
| DAU | 500명 | 3,000명 |
| 1인당 일 광고 노출 | 1.5회 | 2.5회 |
| 월간 광고 노출 | 22,500회 | 225,000회 |
| 평균 eCPM | 3,000원 | 5,000원 |
| **월 예상 수익** | **약 67,500원** | **약 1,125,000원** |

> **참고**: 전기차 관련 광고는 타겟팅 가치가 높아(자동차, 에너지, 보험 등 고단가 광고주) 일반적인 유틸리티 앱 대비 eCPM이 높을 수 있어요.

## 6. 앱인토스 정책 체크

### 위반 가능 항목 점검

| 정책 항목 | 상태 | 설명 |
|-----------|------|------|
| 절대 금지 서비스 (가상자산, 금융중개, 사행성, 데이팅) | 해당 없음 | 충전소 정보 서비스로 금지 항목에 해당하지 않아요 |
| 다크패턴 금지 | 준수 | 광고 닫기 버튼 명확히 제공, 기만적 UX 미사용 |
| 외부 앱 설치 유도 금지 | 주의 필요 | 길찾기 기능에서 외부 네비 앱으로 이동 시, "앱 설치 유도"가 아닌 "기존 설치된 앱 열기"로 구현해야 해요 |
| 외부 결제 페이지 이동 금지 | 해당 없음 | 결제 기능이 없어요 |
| TDS 디자인 필수 | 준수 | `@toss/tds-mobile` 컴포넌트로 전체 UI 구성 |
| 다크모드 미지원 | 준수 | 라이트 모드 전용으로 개발 |
| 토스 로그인만 사용 | 준수 | 별도 로그인 없이 사용 가능 (즐겨찾기는 로컬 저장) |
| 생성형 AI 정책 | 해당 없음 | AI 기능을 사용하지 않아요 |
| 민감 콘텐츠 | 해당 없음 | 성적, 폭력적, 범죄 조장 콘텐츠 없음 |
| 개인정보보호법/정보통신망법 | 준수 | 위치 정보만 사용하며, 서버에 개인정보를 저장하지 않아요 |
| 중복 앱 방지 | 준수 | 기존 토스 내 유사 서비스가 없어요 |
| UX Writing 해요체 | 준수 | 모든 안내 문구를 해요체(비격식 존대체)로 작성해요 |
| Primary Color | 준수 | `#3182F6` (토스 블루) 적용 |

### 필요 권한

| 권한 | 용도 | 필수/선택 | 안내 문구 |
|------|------|-----------|-----------|
| **위치 정보 (GPS)** | 현재 위치 기반 주변 충전소 검색 | 필수 | "주변 충전소를 찾기 위해 위치 정보가 필요해요" |
| **클립보드 접근** | 충전소 주소 복사 기능 | 선택 | 사용자 명시적 액션(복사 버튼 탭) 시에만 사용 |

### 심사 대비 체크리스트

- [ ] TDS 컴포넌트 사용 비율 90% 이상
- [ ] 모든 화면에서 토스 블루(`#3182F6`) 프라이머리 컬러 적용
- [ ] 해요체 UX Writing 전체 적용
- [ ] 위치 정보 수집 동의 안내 화면 구현
- [ ] 길찾기 기능: 외부 앱 "설치 유도"가 아닌 "열기" 방식 확인
- [ ] 광고 닫기 버튼 명확성 확인 (다크패턴 금지)
- [ ] 다크모드 강제 비활성화 처리
- [ ] mTLS 인증서 보안 관리 (2차 서버 구축 시)
- [ ] 앱 설명, 스크린샷, 개인정보처리방침 문서 준비

## 7. MVP 개발 일정 (2주 기준)

### 1주차: 기반 구축 + 핵심 기능

| 일차 | 작업 내용 | 산출물 |
|------|-----------|--------|
| **Day 1** (월) | 프로젝트 초기 설정 (`granite.config.ts`, TDS 설치, 라우팅 구조), 공공데이터포털 API 키 발급, Kakao Maps API 키 발급 | 프로젝트 boilerplate, API 키 |
| **Day 2** (화) | 환경부 충전소 API 연동 (getChargerInfo, getChargerStatus), React Query 기반 데이터 페칭 레이어 구현 | API 서비스 모듈 |
| **Day 3** (수) | 홈 화면 - 지도 뷰 구현 (Kakao Maps 연동, 마커 렌더링, 현재 위치 표시) | 지도 화면 완성 |
| **Day 4** (목) | 충전소 마커 상태별 색상 적용, 필터 칩(충전기 타입) 구현, 마커 탭 시 바텀시트 | 지도 인터랙션 완성 |
| **Day 5** (금) | 목록 화면 구현 (거리순 정렬, TDS ListRow, 무한 스크롤), 지도/리스트 뷰 전환 | 목록 화면 완성 |

### 2주차: 상세 기능 + 광고 연동 + QA

| 일차 | 작업 내용 | 산출물 |
|------|-----------|--------|
| **Day 6** (월) | 충전소 상세 화면 구현 (기본 정보, 충전기 현황, 상태 뱃지, 미니 지도) | 상세 화면 완성 |
| **Day 7** (화) | 길찾기 연동 (카카오맵/네이버맵/티맵 딥링크), 전화 연결, 주소 복사 기능 | 외부 연동 완성 |
| **Day 8** (수) | 검색 화면 구현 (주소/충전소명 검색, 최근 검색어, 자동완성) | 검색 기능 완성 |
| **Day 9** (목) | 앱인토스 인앱광고(IAA) 연동 (전면형 광고 배치, 약관 동의, 정산 정보 입력) | 광고 연동 완성 |
| **Day 10** (금) | 전체 QA (디바이스 테스트, TDS 디자인 심사 기준 점검, 엣지케이스 처리, 성능 최적화), 앱인토스 콘솔 심사 제출 | 심사 제출 완성 |

### 마일스톤 요약

```
Week 1: [프로젝트 셋업] -> [API 연동] -> [지도 뷰] -> [필터/인터랙션] -> [목록 뷰]
Week 2: [상세 화면] -> [길찾기/연동] -> [검색] -> [광고 연동] -> [QA/심사 제출]
```

### 리스크 및 대응

| 리스크 | 영향도 | 대응 방안 |
|--------|--------|-----------|
| 환경부 API 호출 제한 (1,000건/일) 초과 | 높음 | 프론트엔드 캐싱(React Query 5분 TTL), 지역 코드 기반 필터링으로 호출 최소화. 운영 계정 신청으로 한도 확대 |

| 앱인토스 심사 반려 | 중간 | TDS 가이드 철저히 준수, 심사 제출 전 자체 체크리스트 검증 |
| 위치 권한 거부 시 | 낮음 | 주소 검색 기반 폴백, 위치 권한 재요청 안내 |

## 8. 경쟁력 & 차별점

### 기존 경쟁 서비스 분석

| 서비스 | 플랫폼 | 장점 | 단점 |
|--------|--------|------|------|
| **EV Infra** | iOS/Android 앱 | 충전소 DB가 풍부, 사용자 리뷰 | 별도 앱 설치 필요, 광고 과다 |
| **충전인프라 (환경부)** | 웹/앱 | 공식 데이터, 신뢰성 높음 | UI/UX 불편, 느린 로딩 |
| **카카오맵/네이버맵** | 앱 내 기능 | 지도 앱 내 통합, 경로 탐색 | 충전기 실시간 상태 제공 미흡, 전기차 특화 기능 부족 |
| **차지비 (ChargeB)** | iOS/Android 앱 | 충전 결제 통합 | 별도 앱 설치 필요, 특정 사업자 편향 |

### 충전어디의 차별점

| 차별점 | 설명 |
|--------|------|
| **설치 불필요** | 토스 앱 내에서 바로 사용할 수 있어요. 2,200만+ 토스 사용자가 잠재 고객이에요 |
| **실시간 상태 특화** | 단순 위치 정보가 아닌 충전기 상태(충전대기/충전중/고장)를 핵심 기능으로 제공해요 |
| **빠른 접근성** | 토스 앱 열기 -> "충전어디" 검색 -> 바로 주변 충전소 확인. 3단계 이내로 원하는 정보를 얻을 수 있어요 |
| **깔끔한 UX** | TDS 기반의 토스 네이티브급 UI/UX. 토스 사용자에게 익숙한 디자인이에요 |
| **광고 최소화** | 핵심 플로우(지도/목록)에는 광고를 넣지 않아 충전소를 급하게 찾을 때 방해받지 않아요 |
| **토스 생태계 시너지** | 토스 결제, 토스 보험 등과의 향후 연계 가능성이 있어요 (충전 결제, 자동차 보험 연동 등) |
| **경량화** | WebView 기반으로 앱 용량 부담이 없고, 토스 앱 업데이트 없이 서비스 개선이 가능해요 |

### 성장 전략

```
Phase 1 (MVP): 주변 충전소 찾기 + 실시간 상태 확인 -> 전기차 오너 핵심 니즈 해결
Phase 2 (확장): 즐겨찾기 + 알림 + 요금 비교 -> 리텐션 강화, DAU 성장
Phase 3 (플랫폼): 사용자 리뷰 + 혼잡도 통계 + 경로 플래너 -> 전기차 종합 정보 플랫폼
```
