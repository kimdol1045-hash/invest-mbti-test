# 핫나우(HotNow) - 앱인토스 미니앱 기획문서

---

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| **앱 이름 (한글)** | 핫나우 |
| **앱 이름 (영문)** | HotNow |
| **한줄 소개** | 지금 이 순간, 세상의 관심사를 한눈에 보여드려요 |
| **카테고리** | 정보/트렌드 |
| **Primary Color** | `#3182F6` (토스 블루) |

### 타겟 사용자
- **1차 타겟**: "지금 뭐가 떠?" 궁금한 20~30대 정보 소비자
- **2차 타겟**: 투자 시그널을 찾는 주식/코인 투자자
- **3차 타겟**: 트렌드 소재가 필요한 콘텐츠 크리에이터
- **핵심 사용 시나리오**: "출근길 토스 열었는데, 지금 뭐가 핫하지?"

### 기획 배경
- 2021년 네이버 실검 폐지 이후 "지금 뭐가 떠?" 정보 욕구가 여전히 존재
- Signal.bz, 이지미 등 대체 서비스는 모두 웹 기반 → 일부러 찾아가야 하는 구조
- 다음(AXZ)이 2026년 3월 '실시간 트렌드' 베타를 재도입하며 시장 재점화
- 토스 MAU 2,677만명, 일 평균 앱 오픈 10회 → 이미 매일 여는 앱 안에 트렌드 콘텐츠를 제공하면 접근성이 압도적

## 2. 핵심 기능

### MVP 1차

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **실시간 인기 검색어 TOP 20** | 3개 소스 통합 트렌드 점수 기반 순위 리스트 | P0 |
| **지역 탭** | 전체 / 한국 / 해외 탭 전환 | P0 |
| **카테고리 필터** | 전체, 기술, 경제, 스포츠, 엔터, 정치, 생활 (수평 스크롤 pill) | P0 |
| **키워드 상세 (바텀시트)** | AI 요약, 24시간 검색량 추이, 관련 키워드, 소스별 점수 | P0 |
| **자동/수동 새로고침** | 5분 자동 갱신 + 수동 새로고침 버튼 | P1 |
| **더 알아보기** | 네이버/구글/유튜브 검색 링크 (인앱 웹뷰) | P1 |

### 확장 2차

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **푸시 알림** | 모닝 브리핑, 급상승 알림, 이브닝 요약 | P2 |
| **관심 카테고리 설정** | 사용자별 관심 카테고리 선택 및 우선 정렬 | P2 |
| **트렌드 예측 게임** | "내일 1위 맞히기" 탭 | P3 |
| **개인화 피드** | 관심사 기반 "나만의 트렌드 피드" | P3 |

## 3. 화면 구성

### 3-1. 화면 목록

#### (1) 메인 화면 — 실시간 인기 검색어 TOP 20

- **상단**: `Navigation` 컴포넌트 - "핫나우" 타이틀, 새로고침 버튼
- **업데이트 시각**: "14:30 기준" 형태로 표시
- **지역 탭**: `SegmentedControl` - "전체" | "한국" | "해외"
- **카테고리 필터**: 수평 스크롤 pill 칩 - 전체, 기술, 경제, 스포츠, 엔터, 정치, 생활
- **키워드 리스트**: 커스텀 `TrendRow` 컴포넌트 반복
  - 순위 번호 (TOP 3 강조: 크고 Bold, 토스 블루)
  - 키워드명
  - 변동 표시: ▲(빨강), ▼(파랑), NEW(토스 블루 Badge), -(회색)
  - 검색량 (만/천 단위 포맷)
  - 카테고리 태그 (작은 pill)
- **하단 안내**: "5분마다 업데이트 · Google Trends, Naver 등"
- **AI 사용 고지**: "AI가 키워드를 분류하고 요약해요" 안내

#### (2) 키워드 상세 (바텀시트)

키워드 터치 시 바텀시트로 상세 진입.

- **헤더**: 키워드명 + 현재 순위 Badge
- **검색량 추이**: 최근 24시간 라인 차트 (간단한 SVG 차트)
- **AI 요약 섹션**: "왜 떴나?" + Haiku 4.5 생성 2~3문장 요약
  - AI 생성물 표시: "AI가 요약했어요" 라벨
- **관련 키워드**: 연관 키워드 태그 (터치 시 해당 키워드 상세로 이동)
- **소스별 검색량**: 각 데이터 소스별 정규화 점수 프로그레스 바
- **더 알아보기**: 네이버/구글/유튜브 검색 링크 (인앱 웹뷰)

### 3-2. 화면 플로우

```
[토스 앱]
    |
    v
[메인 화면 - TOP 20 리스트]
    |
    ├── 지역 탭 전환 (전체/한국/해외)
    ├── 카테고리 필터 (기술/경제/스포츠/...)
    |
    v  (키워드 터치)
[키워드 상세 - 바텀시트]
    |
    ├── AI 요약 확인
    ├── 관련 키워드 → 다른 키워드 상세
    └── 더 알아보기 → 인앱 웹뷰 (네이버/구글/유튜브)
```

### 3-3. 라우팅 구조 (intoss:// scheme)

| 경로 | 화면 | 설명 |
|------|------|------|
| `intoss://hot-now` | 메인 (TOP 20) | 메인 진입점 |

> 키워드 상세는 바텀시트로 처리하여 별도 라우트 없이 동일 페이지에서 오버레이 형태로 표시

## 4. 기술 스택

### 4-1. 프론트엔드 (앱인토스 미니앱)

| 항목 | 기술 | 비고 |
|------|------|------|
| **플랫폼** | WebView (앱인토스) | |
| **프레임워크** | React 18 + TypeScript | `@apps-in-toss/web-framework@^2.0.1` |
| **디자인 시스템** | `@toss/tds-mobile@^2.2.1` | 심사 필수 |
| **빌드 도구** | Vite | 기본 설정 |
| **상태 관리** | React useState/useReducer | 경량 MVP |
| **HTTP 클라이언트** | fetch API | Supabase Edge Functions 호출 |
| **차트** | SVG 직접 구현 | 외부 차트 라이브러리 없이 경량화 |

### 4-2. 백엔드 (별도 구축)

| 레이어 | 기술 | 비고 |
|--------|------|------|
| **API** | Supabase Edge Functions (Deno) | REST API |
| **DB** | Supabase PostgreSQL | 트렌드 데이터 저장 |
| **데이터 수집** | Python 3.11+ (asyncio) | 8개 소스 크롤러 |
| **AI** | Claude Haiku 4.5 | 정규화/분류/요약/필터링 |
| **벡터DB** | Qdrant Cloud | 생태계 연동 |

### 4-3. API 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/trending` | GET | 실시간 TOP 20 (region, category 파라미터) |
| `/trending/:id` | GET | 키워드 상세 (AI 요약, 관련 키워드, 소스별 점수) |
| `/history/:keyword` | GET | 24시간 검색량 추이 |

### 4-4. 데이터 저장

| 저장소 | 용도 | 기술 |
|--------|------|------|
| **네이티브 스토리지** | 관심 카테고리 설정 | 앱인토스 `get/set` API |
| **클라이언트 캐시** | API 응답 캐싱 (TTL 5분) | 메모리 캐시 |

## 6. 앱인토스 정책 준수

### 6-1. 등록 불가 서비스 해당 여부

| 금지 항목 | 해당 여부 |
|-----------|-----------|
| 가상자산/NFT | 해당 없음 |
| 현금/자산 교환 | 해당 없음 |
| 사행성/도박 | 해당 없음 |
| 금융 상품 중개/판매 | 해당 없음 |
| 투자 자문 | 해당 없음 |
| 소셜/데이팅 | 해당 없음 |

> **결론: 정책 위반 사항 없음.** 정보/트렌드 카테고리로 등록 가능.

### 6-2. 주의 사항

| 항목 | 대응 |
|------|------|
| **AI 사용 고지** | "AI가 키워드를 분류하고 요약해요" 안내 문구 상시 표시 |
| **AI 생성물 표시** | AI 요약 영역에 "AI가 요약했어요" 라벨 표시 |
| **외부 링크** | "더 알아보기"는 인앱 웹뷰로 열기, 필수가 아닌 부가 기능 배치 |
| **민감 키워드 필터링** | 4단계 필터링 시스템 (블랙리스트 → AI 판단 → 선거 기간 → 수동 모니터링) |
| **다크패턴 금지** | 강제 체류 유도, 기만적 UI 패턴 없음 |
| **푸시 글자 수** | 제목 7자 이하, 내용 25자 이하 |

### 6-3. 심사 대비 체크리스트

- [ ] TDS 컴포넌트 사용 (`@toss/tds-mobile`)
- [ ] 토스 로그인만 사용 (자체 로그인 없음)
- [ ] 다크모드 미지원 (라이트 모드 전용)
- [ ] Primary Color `#3182F6` 적용
- [ ] 해요체 UX Writing
- [ ] 접근성 명도 대비 4.5:1 이상
- [ ] AI 사용 고지 필수 표시
- [ ] AI 생성물 가시적 표시
- [ ] 외부 앱 설치 유도 없음
- [ ] 외부 결제 페이지 이동 없음
- [ ] 다크패턴 없음
