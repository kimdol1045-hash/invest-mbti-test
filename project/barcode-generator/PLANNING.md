# 바코드/QR 생성기 - 앱인토스 미니앱 기획문서

---

## 1. 앱 개요

### 앱 이름
- **한글**: 바로코드
- **영문**: BaroCode

### 한줄 소개
텍스트, URL, 연락처, 와이파이 등을 입력하면 바코드와 QR코드를 즉시 생성하고 저장/공유할 수 있는 초간단 유틸리티 미니앱이에요.

### 타겟 사용자
| 타겟 | 설명 |
|------|------|
| **소상공인/자영업자** | 매장 와이파이 QR, 결제 링크 QR, 상품 바코드 등을 빠르게 만들고 싶은 사업자 |
| **직장인/학생** | 명함 QR, URL 공유 QR, 문서 링크 QR 등을 간편하게 생성하고 싶은 일반 사용자 |
| **행사/이벤트 기획자** | 참가 등록 QR, 안내 URL QR 등을 빠르게 만들어야 하는 기획자 |
| **일상 사용자** | 토스 앱 안에서 별도 앱 설치 없이 바코드/QR을 바로 만들고 싶은 모든 토스 사용자 |

**핵심 가치**: 별도 앱 설치 없이 토스 안에서 3초 만에 바코드/QR 생성 완료

## 3. 화면 구성

### 3-1. 화면 목록

#### [홈 화면] - 메인 진입점
- **TDS Navigation**: 상단 "바로코드" 타이틀
- **TDS Tab**: "QR코드" / "바코드" 탭 전환
- **입력 타입 그리드**: TDS ListRow 기반 프리셋 목록
  - URL, 텍스트, 연락처, 와이파이, 이메일, 전화번호, SMS, 위치
  - 각 항목에 아이콘 + 라벨 표시
- **최근 생성 섹션**: TDS ListHeader("최근 생성") + ListRow 리스트 (최대 3개 미리보기)
- **하단 AdMob 배너 광고**: 화면 하단 고정

#### [생성 화면] - 입력 및 즉시 생성
- **TDS Navigation**: 뒤로가기 + "QR코드 만들기" / "바코드 만들기" 타이틀
- **입력 폼**: 선택한 타입에 맞는 TDS TextField/TextArea
  - URL 타입: URL 입력 필드 1개
  - 연락처 타입: 이름, 전화번호, 이메일, 회사명 필드
  - 와이파이 타입: SSID, 비밀번호, 암호화 방식(WPA/WPA2/WEP/없음) 선택
  - 텍스트 타입: 자유 텍스트 입력 (TextArea)
- **실시간 미리보기**: 입력과 동시에 QR/바코드 미리보기 렌더링
- **TDS BottomCTA**: "코드 생성하기" 버튼 (#3182F6)

#### [결과 화면] - 생성 완료 및 액션
- **TDS Navigation**: 뒤로가기 + "완성!" 타이틀
- **코드 이미지 영역**: 생성된 QR/바코드 대형 표시 (중앙 정렬)
- **입력 내용 요약**: TDS Paragraph로 원본 입력값 표시
- **액션 버튼 영역**:
  - TDS Button "이미지 저장하기" (Primary, #3182F6)
  - TDS Button "공유하기" (Secondary)
  - TDS Button "클립보드 복사" (Tertiary)
- **하단 AdMob 배너 광고**

#### [히스토리 화면] - 생성 이력 관리
- **TDS Navigation**: 뒤로가기 + "생성 히스토리" 타이틀
- **TDS ListRow 리스트**: 생성 일시, 타입 아이콘, 입력값 미리보기, QR/바코드 썸네일
- **스와이프 삭제**: 개별 항목 삭제
- **TDS ListHeader**: "전체 삭제" 액션 버튼
- **빈 상태**: 아직 생성한 코드가 없을 때 안내 메시지

### 3-2. 화면 플로우

```
┌─────────────┐
│   홈 화면    │
│ (타입 선택)  │
└──────┬──────┘
       │ 타입 선택
       v
┌─────────────┐
│  생성 화면   │
│ (입력 폼)   │
└──────┬──────┘
       │ "코드 생성하기" 탭
       v
┌─────────────┐
│  결과 화면   │──→ 이미지 저장 / 공유 / 복사
│ (코드 표시)  │
└──────┬──────┘
       │ 뒤로가기
       v
┌─────────────┐
│   홈 화면    │──→ "최근 생성" 탭 → 히스토리 화면
└─────────────┘
```

**주요 네비게이션 경로**:
1. 홈 → 타입 선택 → 생성 화면 → 결과 화면 → (저장/공유) → 홈
2. 홈 → 최근 생성 더보기 → 히스토리 화면 → 항목 선택 → 결과 화면
3. 결과 화면 → 뒤로가기 → 생성 화면 (입력값 유지)

**라우팅 구조** (`intoss://` scheme):
```
intoss://barocode              → 홈 화면
intoss://barocode/create       → 생성 화면 (?type=url&mode=qr)
intoss://barocode/result       → 결과 화면
intoss://barocode/history      → 히스토리 화면
```

## 4. 기술 스택

### 프론트엔드

| 항목 | 기술 | 비고 |
|------|------|------|
| **프레임워크** | `@apps-in-toss/web-framework` (React) | WebView 미니앱 |
| **디자인 시스템** | `@toss/tds-mobile` | TDS 필수 사용 |
| **QR코드 생성** | `qrcode` (npm) 또는 `qrcode.react` | 클라이언트 사이드 QR 생성 |
| **바코드 생성** | `JsBarcode` (npm) | Code128, EAN-13 등 지원 |
| **이미지 처리** | `html2canvas` 또는 Canvas API | 코드 이미지를 PNG로 변환 |
| **상태 관리** | React useState/useReducer | MVP 단계에서는 경량 관리 |
| **빌드 도구** | Vite | `granite.config.ts` 기반 |

### 필요한 외부 API / 네이티브 API

| API | 용도 | 필수 여부 |
|-----|------|-----------|
| **네이티브 저장소** (`storage.set/get`) | 히스토리 데이터 로컬 저장 | 필수 |
| **공유 API** (`share`) | 생성된 이미지/링크 공유 | 필수 |
| **Base64 파일 저장** | PNG 이미지를 디바이스에 저장 | 필수 |
| **AdMob 광고** (SDK v1.11.0+) | 배너/전면/보상형 광고 노출 | 필수 (수익화) |
| **클립보드 API** | 코드 내용 복사 | 선택 |
| **카메라 API** (`openCamera`) | QR 스캔 기능 (2차) | 2차 확장 |

### 데이터 저장 방식

| 데이터 | 저장소 | 형식 |
|--------|--------|------|
| 생성 히스토리 | 네이티브 스토리지 (`storage.set/get`) | JSON 배열 |
| 즐겨찾기 목록 | 네이티브 스토리지 | JSON 배열 |
| 생성된 이미지 | 디바이스 갤러리 (Base64 파일 저장) | PNG |
| 사용자 설정 | 네이티브 스토리지 | JSON 객체 |

**히스토리 데이터 스키마 예시**:
```typescript
interface HistoryItem {
  id: string;           // UUID
  type: 'qr' | 'barcode';
  inputType: 'url' | 'text' | 'contact' | 'wifi' | 'email' | 'phone' | 'sms' | 'location';
  inputData: Record<string, string>;
  createdAt: string;    // ISO 8601
  thumbnailBase64?: string;  // 썸네일 이미지 (압축)
}
```

**서버 불필요**: MVP 단계에서는 모든 처리가 클라이언트 사이드에서 이루어지므로 별도 서버/mTLS 설정이 필요하지 않아요. 히스토리 데이터는 네이티브 스토리지에 로컬 저장해요.

| 정책 항목 | 위반 여부 | 상세 |

## 8. 경쟁력 & 차별점

### 기존 서비스 대비 분석

| 비교 항목 | 기존 QR 생성 앱 | 바로코드 (본 서비스) |
|-----------|-----------------|---------------------|
| **접근성** | 별도 앱 설치 필요 | 토스 앱 내에서 바로 사용, 설치 불필요 |
| **사용자 기반** | 0에서 시작 | 토스 3,000만 사용자 즉시 접근 가능 |
| **UX 일관성** | 앱마다 상이 | TDS 기반 토스 네이티브 경험 |
| **공유 편의성** | OS 공유 시트만 지원 | 토스 네이티브 공유 + 이미지 저장 |
| **광고 경험** | 과도한 광고, 다크패턴 | 절제된 광고, 토스 정책 준수 |
| **속도** | 앱 실행 → 생성까지 오래 걸림 | 타입 선택 → 입력 → 생성 3단계, 3초 완료 |
| **신뢰도** | 개인 개발자 앱 신뢰 이슈 | 토스 플랫폼 심사 통과, 높은 신뢰도 |

### 핵심 차별점

1. **제로 프릭션(Zero Friction)**: 토스 앱 안에서 별도 설치/회원가입 없이 즉시 사용. 토스 로그인으로 자동 인증되므로 사용자 경험이 매끄러워요.

2. **토스 네이티브 경험**: TDS 디자인 시스템을 사용해 토스 앱과 동일한 UI/UX를 제공해요. 사용자가 새로운 앱을 배울 필요가 없어요.

3. **빠른 생성 플로우**: 타입 선택 → 입력 → 즉시 생성의 3단계 플로우로 3초 안에 코드를 만들 수 있어요. 실시간 미리보기로 결과를 바로 확인할 수 있어요.

4. **건강한 광고 경험**: 기존 QR 생성 앱들의 과도한 광고와 다크패턴 없이, 앱인토스 정책을 준수하는 절제된 광고 배치로 사용자 경험을 보호해요.

5. **대규모 사용자 풀 즉시 접근**: 토스 앱 3,000만 사용자에게 발견될 수 있어요. Smart Message, 세그먼트 타겟팅 등 토스 마케팅 도구를 활용할 수 있어요.

6. **서버 불필요 아키텍처**: 모든 코드 생성이 클라이언트에서 처리되므로 서버 운영 비용이 없어요. MVP를 최소 비용으로 빠르게 출시할 수 있어요.
