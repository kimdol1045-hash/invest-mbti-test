## 1. 앱 개요

### 앱 이름
- **한글**: 무료전시 찾기
- **영문**: FreeShow (프리쇼)

### 한줄 소개
> 내 주변 무료 전시, 공연, 축제를 한눈에 찾아보세요.

### 서비스 설명
공공데이터포털, 서울열린데이터광장, 문화포털 등 지자체 공공데이터 API를 활용하여 전국의 무료 문화행사(전시, 공연, 축제, 체험 프로그램 등) 정보를 수집하고, 사용자 위치 기반으로 가까운 행사부터 보여주는 앱인토스 미니앱이에요.

### 타겟 사용자
| 구분 | 설명 |
|------|------|
| **핵심 타겟** | 20~30대 문화생활에 관심 있는 토스 사용자 |
| **확장 타겟** | 주말 나들이 계획을 세우는 가족 단위 사용자 |
| **니치 타겟** | 전시/공연 정보를 자주 찾는 문화 애호가 |

### 핵심 가치
- **무료 행사만 모아서** 비용 부담 없이 문화생활을 즐길 수 있어요
- **공공데이터 기반**으로 신뢰할 수 있는 정보를 제공해요
- **위치 기반 추천**으로 가까운 행사를 쉽게 찾을 수 있어요

## 3. 화면 구성

### 화면 목록

#### 3-1. 홈 화면 (`/pages/index.tsx`)
- **라우트**: `intoss://free-show`
- **구성 요소**:
  - `Navigation` - 상단 앱 타이틀 "무료전시 찾기"
  - `Tab` - 카테고리 탭 (전체 / 전시 / 공연 / 축제 / 체험)
  - `SearchField` - 키워드 검색 입력
  - `ListRow` 반복 - 행사 목록 (썸네일, 제목, 기간, 장소, 거리)
  - 하단 `Banner Ad` - AdMob 배너 광고
- **동작**:
  - 앱 진입 시 위치 권한 요청 → 승인 시 가까운 순 정렬, 거부 시 최신순 정렬
  - 스크롤 시 무한 로딩 (페이지네이션)
  - 탭 전환 시 카테고리 필터 적용

#### 3-2. 행사 상세 화면 (`/pages/detail/[id].tsx`)
- **라우트**: `intoss://free-show/detail/{id}`
- **구성 요소**:
  - `Navigation` - 뒤로가기 + 공유 버튼
  - 대표 이미지 (가로 풀사이즈)
  - `Badge` - 카테고리 태그 (전시/공연/축제 등)
  - `Badge` - 진행 상태 (진행중/예정/종료)
  - `Paragraph` - 행사 제목, 기간, 시간, 장소, 상세 설명
  - `ListRow` - 장소 정보 (주소, 전화번호)
  - 지도 미리보기 (좌표 기반)
  - `BottomCTA` - "길찾기" 버튼 (외부 지도앱 딥링크)
- **동작**:
  - 상세 정보 API 호출 및 렌더링
  - 공유 버튼 클릭 시 토스 공유 기능 호출
  - 길찾기 버튼 클릭 시 지도 앱 연동

#### 3-3. 지역 필터 화면 (`/pages/region.tsx`)
- **라우트**: `intoss://free-show/region`
- **구성 요소**:
  - `Navigation` - "지역 선택"
  - `ListRow` 반복 - 시/도 목록 (서울, 경기, 부산 등 17개 시도)
  - 선택 시 하위 시/군/구 목록 표시
  - `BottomCTA` - "적용하기" 버튼
- **동작**:
  - 지역 선택 후 홈 화면으로 돌아가며 해당 지역 행사만 필터링

#### 3-4. 북마크 화면 (`/pages/bookmark.tsx`) [2차]
- **라우트**: `intoss://free-show/bookmark`
- **구성 요소**:
  - `Navigation` - "찜한 행사"
  - `ListRow` 반복 - 북마크한 행사 목록
  - 빈 상태 시 안내 메시지
- **동작**:
  - 네이티브 스토리지에 저장된 북마크 행사 목록 표시
  - 스와이프 삭제 가능

### 화면 플로우

```
[홈 화면]
  ├─ 검색 입력 → 검색 결과 (홈 화면 내 필터링)
  ├─ 카테고리 탭 선택 → 해당 카테고리 목록
  ├─ 지역 필터 버튼 → [지역 필터 화면] → 적용 → [홈 화면]
  ├─ 행사 항목 클릭 → [행사 상세 화면]
  │     ├─ 공유 버튼 → 토스 공유
  │     ├─ 길찾기 버튼 → 외부 지도앱
  │     └─ 뒤로가기 → [홈 화면]
  └─ 북마크 아이콘 → [북마크 화면] (2차)
```

### 외부 API (공공데이터)

#### 주요 API 목록

| API | 제공처 | 용도 | 엔드포인트/데이터셋 |
|-----|--------|------|---------------------|
| **한눈에보는문화정보조회서비스** | 한국문화정보원 (공공데이터포털) | 전국 공연/전시/문화행사 통합 조회 | `data.go.kr/data/15138937` |
| **공연전시정보조회서비스** | 문화포털 | 기간별/지역별 공연전시 검색 | `culture.go.kr/openapi/rest/publicperformancedisplays/period` |
| **전국문화축제표준데이터** | 공공데이터포털 | 전국 문화축제 정보 | `data.go.kr/data/15013104` |
| **전국공연행사정보표준데이터** | 공공데이터포털 | 전국 공연/행사 정보 | `data.go.kr/data/15013106` |
| **문화예술공연(통합)** | 문화체육관광부 | 문화예술 공연 통합 정보 | `data.go.kr/data/15121487` |
| **서울시 문화행사 정보** | 서울열린데이터광장 | 서울 지역 문화행사 | `data.seoul.go.kr OA-15486` |
| **경기도 문화행사 현황** | 경기데이터드림 | 경기도 지역 문화행사 | `data.gg.go.kr` |

#### API 활용 전략

1. **1순위 (메인)**: 한눈에보는문화정보조회서비스 - 전국 단위 통합 데이터, 공연/전시/문화예술/관광 등 폭넓은 카테고리 지원
2. **2순위 (보조)**: 공연전시정보조회서비스 - 기간별, 좌표(GPS) 기반 검색 지원으로 위치 기반 탐색에 적합
3. **3순위 (축제 특화)**: 전국문화축제표준데이터 - 축제 카테고리 전용 데이터
4. **4순위 (지역 보강)**: 서울시/경기도 등 각 지자체 데이터로 지역별 커버리지 보강

#### API 요청 예시 (공연전시정보조회서비스)

```
GET http://www.culture.go.kr/openapi/rest/publicperformancedisplays/period
?from=20260301
&to=20260401
&cPage=1
&rows=20
&place=
&gpsxfrom=126.9
&gpsyfrom=37.5
&gpsxto=127.1
&gpsyto=37.6
&keyword=
&sortStdr=1
&serviceKey={발급받은_서비스키}
```

#### API 주요 응답 필드

| 필드 | 설명 |
|------|------|
| `title` | 행사 제목 |
| `period` | 행사 기간 (시작일~종료일) |
| `eventSite` | 행사 장소명 |
| `charge` | 관람료 (무료 여부 필터링 기준) |
| `area` | 지역 |
| `gpsX`, `gpsY` | 좌표 (위치 기반 거리 계산) |
| `thumbnail` | 썸네일 이미지 URL |
| `url` | 상세 페이지 URL |

### 데이터 저장

| 구분 | 기술 | 용도 |
|------|------|------|
| **캐시** | React Query 인메모리 캐시 | API 응답 캐싱 (5분 stale time) |
| **북마크** | 네이티브 스토리지 (`get`/`set`/`clear`) | 찜한 행사 ID 목록 저장 |
| **지역 설정** | 네이티브 스토리지 | 선택한 지역 필터 유지 |
| **서버 DB** | 자체 서버 불필요 (공공데이터 직접 호출) | - |

### 서버 구성

- **자체 서버 최소화**: 공공데이터 API를 클라이언트에서 직접 호출하지 않고, 경량 프록시 서버를 통해 호출
- **프록시 서버 역할**:
  - API 키 보호 (클라이언트에 서비스키 노출 방지)
  - 여러 공공데이터 API 응답을 통합/정규화
  - 무료 행사만 필터링하여 전달
  - 응답 캐싱으로 API 호출 횟수 최적화
- **서버 기술**: Node.js + Express (또는 Cloudflare Workers)
- **mTLS**: 토스 서버 통신 시 mTLS 인증서 기반 통신 필수

## 5. 수익화 전략

### IAA (In-App Advertising) 기반

앱인토스 SDK v1.11.0부터 지원하는 In-app Ads 2.0을 활용해요.

| 광고 유형 | 위치 | 노출 조건 | 예상 수익 |
|-----------|------|-----------|-----------|
| **배너 광고** | 홈 화면 하단 고정 | 항상 노출 | 기본 수익원 |
| **전면 광고** | 행사 상세 → 홈 복귀 시 | 3회 상세 조회마다 1회 | 중간 수익원 |
| **보상형 광고** | 북마크 5개 초과 저장 시 | 사용자 선택적 시청 | 보조 수익원 |

### 광고 플랫폼
- **토스 Ads + Google AdMob** 하이브리드 (SDK v1.11.0+ 기본 제공)

### 수수료 구조
- 인앱 광고 수수료: CBT 기간 15% (현재 0% 프로모션)
- 인앱결제 미사용 (무료 서비스이므로 결제 불필요)

### 수익화 원칙
- 사용자 경험을 해치지 않는 선에서 광고 노출
- 광고 때문에 핵심 콘텐츠가 가려지지 않도록 배치
- 다크패턴(강제 광고 시청, 닫기 버튼 은닉 등) 절대 금지

### 향후 수익 확장 (2차)
- 유료 행사 티켓 예매 연동 시 토스페이 결제 수수료
- 문화공간/갤러리 제휴 광고 (프리미엄 리스트 상단 노출)

| 접근성 | 명도 대비 4.5:1 이상 준수 |
| Navigation | TDS `Navigation` 컴포넌트 사용 |

### 필요 권한

| 권한 | 용도 | 필수 여부 |
|------|------|-----------|
| `location` | 사용자 위치 기반 가까운 행사 탐색 | 선택 (거부 시 최신순 정렬로 대체) |

### 보안 심사 대응

| 항목 | 대응 방안 |
|------|-----------|
| 토스 로그인 | OAuth 2.0 기반 토스 로그인 사용 (자체/제3자 로그인 없음) |
| mTLS | 프록시 서버 ↔ 토스 서버 간 mTLS 인증서 기반 통신 |
| API 키 보호 | 공공데이터 API 서비스키를 프록시 서버에서만 관리 |
| 개인정보 | 위치 정보 외 별도 개인정보 수집하지 않음 |

## 8. 경쟁력 & 차별점

### 기존 서비스와의 비교

| 비교 항목 | 무료전시 찾기 (FreeShow) | 문화포털 앱 | 서울문화포털 | 네이버 전시/공연 |
|-----------|-------------------------|-------------|-------------|-----------------|
| **무료 행사 전용** | O (핵심 특화) | X (유/무료 혼합) | X (유/무료 혼합) | X (유/무료 혼합) |
| **위치 기반 추천** | O (GPS 거리순) | X | 제한적 | 제한적 |
| **토스 앱 내 접근** | O (미니앱) | X (별도 앱) | X (웹) | X (별도 앱) |
| **전국 통합 데이터** | O (다중 API 통합) | O | X (서울만) | O |
| **간편한 접근성** | O (토스 앱 내 즉시 실행) | 앱 설치 필요 | 브라우저 접속 | 앱 설치 필요 |

### 핵심 차별점

#### 1. "무료" 행사에 집중한 큐레이션
- 유료 행사와 섞이지 않아 선택 피로도 감소
- "무료인데 이렇게 좋은 행사가 있어요" 라는 발견의 즐거움 제공
- 경제적 부담 없이 문화생활을 즐기고 싶은 니즈에 정확히 부합

#### 2. 토스 앱 내 원클릭 접근
- 별도 앱 설치 불필요, 토스 앱 내에서 바로 실행
- 토스의 3,000만 사용자 기반에 자연스럽게 노출
- 토스 사용자의 "절약", "혜택 탐색" 성향과 높은 시너지

#### 3. 공공데이터 다중 소스 통합
- 단일 API가 아닌 여러 공공데이터 소스를 통합하여 커버리지 극대화
- 문화포털 + 공공데이터포털 + 지자체 데이터 결합으로 누락 최소화
- 공공데이터 기반이므로 데이터 신뢰성 확보

#### 4. 위치 기반 개인화
- 현재 위치에서 가까운 행사를 자동 추천
- "지금 바로 갈 수 있는 무료 행사" 제안으로 즉시 행동 유도
- 지역별 필터로 여행지/외출지 행사도 미리 탐색 가능

#### 5. 가벼운 기술 구조 = 빠른 시장 진입
- 자체 콘텐츠 생산 불필요 (공공데이터 활용)
- 자체 DB/서버 최소화로 운영 비용 절감
- 2주 내 MVP 출시 가능한 린(Lean) 구조

### 성장 전략

| 단계 | 전략 | 목표 |
|------|------|------|
| **출시 초기** | 토스 Smart Message로 "주말 무료 전시 추천" 푸시 | DAU 1,000 달성 |
| **안정기** | 프로모션 (토스 포인트 리워드), 공유 리워드 | DAU 5,000 달성 |
| **성장기** | 유료 행사 확장, 티켓 예매 연동, 제휴 확대 | MAU 50,000+ |
