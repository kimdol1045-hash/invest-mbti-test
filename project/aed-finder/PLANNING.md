## 1. 앱 개요

### 앱 이름
- **한글**: 심장지킴이
- **영문**: AED Finder

### 한줄 소개
> 내 주변 자동심장충격기(AED) 위치를 실시간으로 찾고, 응급 상황 시 CPR 가이드를 제공하는 공익 미니앱이에요.

### 타겟 사용자
| 구분 | 대상 | 설명 |
|------|------|------|
| 1차 타겟 | 일반 시민 (19세 이상) | 응급 상황 발생 시 AED 위치를 빠르게 찾아야 하는 모든 사람 |
| 2차 타겟 | 안전 관리 담당자 | 건물/시설 관리자, 학교/기업 안전 담당자 |
| 3차 타겟 | 응급처치 교육 관심자 | CPR/AED 사용법을 학습하고 싶은 사용자 |

### 앱 컨셉
- **핵심 가치**: 골든타임 4분 안에 AED를 찾아 생명을 살리는 것
- **슬로건**: "4분의 기적, 당신 곁의 심장지킴이"
- 한국에서 심정지 환자의 생존율은 약 8.7%에 불과하지만, 4분 이내 AED 사용 시 생존율이 70% 이상으로 급증해요
- 토스 앱 사용자 3,000만명에게 생명을 살리는 도구를 제공하는 공익 서비스예요

## 2. 핵심 기능

### MVP 1차 (필수 기능)

#### 2.1 내 주변 AED 지도
- 현재 위치 기반 반경 500m/1km/2km 내 AED 설치 장소 표시
- 지도 위 AED 마커 표시 (설치 장소별 아이콘 구분)
- 마커 탭 시 상세 정보 표시 (설치 장소명, 주소, 층/위치 상세, 관리기관, 연락처)
- GPS 기반 실시간 위치 추적

#### 2.2 AED 상세 정보
- AED 설치 상세 위치 (건물명, 층수, 구체적 위치 설명)
- 관리 기관 정보 및 연락처
- AED 제조사 및 모델명
- 길찾기 연동 (카카오맵/네이버 지도 딥링크)

#### 2.3 CPR 가이드 (응급 상황 대처법)
- **119 신고 바로가기**: 원탭 119 전화 연결
- **심폐소생술(CPR) 단계별 가이드**:
  1. 반응 확인: 양어깨를 두드리며 의식 확인
  2. 119 신고 및 AED 요청
  3. 가슴 압박: 흉골 1/2 아래, 5~6cm 깊이, 분당 100~120회
  4. 인공호흡: 가슴압박 30회 + 인공호흡 2회 반복
  5. AED 사용: 전원 켜기 -> 패드 부착 -> 분석 대기 -> 쇼크 버튼
  6. 심폐소생술 재개: 2분마다 AED 리듬 분석 반복
- 각 단계별 일러스트/애니메이션 가이드
- 가슴 압박 리듬 메트로놈 기능 (분당 100~120회 비프음)

#### 2.4 리스트 뷰 검색
- 현재 위치 기준 가까운 순 AED 목록
- 거리 및 도보 예상 시간 표시
- 주소/장소명 텍스트 검색

### 확장 2차 (추후 개발)

#### 2.5 즐겨찾기
- 자주 방문하는 장소 주변 AED 저장
- 직장/집/학교 등 자주 가는 곳 AED 사전 확인

#### 2.6 AED 정보 제보/업데이트
- 사용자가 AED 위치 정보 오류 신고
- 신규 AED 설치 위치 제보 (사진 첨부)
- 커뮤니티 기반 데이터 보완

#### 2.7 응급처치 퀴즈
- CPR/AED 사용법 학습 퀴즈
- 정답률에 따른 배지 부여
- 토스 포인트 리워드 연계 (프로모션)

#### 2.8 안전 알림
- 주변 AED 점검 필요 알림 (관리자용)
- 심정지 응급 상황 관련 계절별/시간대별 안전 팁 푸시

#### 2.9 오프라인 모드
- 최근 조회한 AED 위치 데이터 로컬 캐싱
- 네트워크 불안정 시에도 마지막 동기화 데이터 표시

## 4. 기술 스택

### 4.1 프론트엔드

| 항목 | 기술 | 설명 |
|------|------|------|
| 플랫폼 | 앱인토스 WebView | `@apps-in-toss/web-framework` |
| 프레임워크 | React 18 + TypeScript | SPA 구조 |
| 빌드 도구 | Vite | 빠른 개발 서버 + 번들링 |
| UI 라이브러리 | `@toss/tds-mobile` | TDS 컴포넌트 필수 사용 |
| 추가 UI | `@toss/tds-mobile-ait` (v1.0.1) | 앱인토스 전용 TDS 확장 |
| 지도 SDK | 카카오맵 JavaScript SDK | 지도 렌더링 + AED 마커 |
| 상태 관리 | React Query (TanStack Query) | API 데이터 캐싱 + 동기화 |
| 라우팅 | 파일 기반 라우팅 | `intoss://` scheme |
| 유틸리티 | `@toss/es-toolkit` | 모던 JS 유틸리티 |

### 4.3 데이터 저장

| 구분 | 방식 | 용도 |
|------|------|------|
| 서버 캐싱 | 자체 백엔드 (Node.js + mTLS) | 공공 API 데이터 캐싱, 사용자 데이터 |
| 로컬 캐싱 | 네이티브 스토리지 (get/set) | 최근 조회 AED, 즐겨찾기, 설정값 |

| 서버 DB | PostgreSQL + PostGIS | AED 위치 좌표 기반 공간 쿼리 |

### 4.4 서버 아키텍처

```
[토스 앱] ──── [앱인토스 WebView] ──── mTLS ──── [자체 API 서버]
                                                      │
                                         ┌────────────┤
                                         ▼            ▼
                                  [공공데이터 API]  [PostgreSQL]
                                  (AED 정보 조회)   (캐싱 DB)
```

- 자체 API 서버: Node.js (Express/Fastify)
- mTLS 인증서: 앱인토스 콘솔에서 발급 (유효기간 390일)
- 엔드포인트: `https://apps-in-toss-api.toss.im/`
- 공공 API 데이터는 서버에서 주기적으로 동기화 (1일 1회)하여 DB에 캐싱

## 5. 수익화 전략

### 5.1 인앱 광고 (IAA) - 주 수익원

| 광고 유형 | 위치 | 노출 시점 | 비고 |
|-----------|------|-----------|------|
| 배너 광고 | 리스트 검색 화면 하단 | 상시 | AdMob 배너 |
| 전면 광고 | 앱 재진입 시 | 5회 사용마다 1회 | AdMob 전면광고, 응급 상황 시 비노출 |
| 보상형 광고 | CPR 퀴즈 결과 후 | 사용자 선택 시 | 토스 포인트 리워드 연계 |

**광고 정책 (공익성 고려):**
- 119 신고 화면, CPR 가이드 진행 중, AED 상세 화면에서는 광고를 절대 노출하지 않아요
- 응급 상황 판단 시 (CPR 가이드 진입 후) 광고 일시 중단
- 사용자 경험을 해치지 않는 범위에서만 광고 노출

### 5.2 수수료 구조

| 항목 | 비율 | 비고 |
|------|------|------|
| 인앱 광고 수수료 | CBT 기간 15% (현재 0%) | 앱인토스 플랫폼 수수료 |
| AdMob 수익 분배 | 개발자 68% / Google 32% | Google AdMob 기본 분배율 |

### 5.3 예상 수익 시뮬레이션 (월 기준)

| 지표 | 보수적 | 중간 | 낙관적 |
|------|--------|------|--------|
| DAU | 5,000 | 20,000 | 50,000 |
| 배너 CPM | 800원 | 1,200원 | 1,500원 |
| 전면 광고 CPM | 5,000원 | 8,000원 | 12,000원 |
| 월 예상 수익 | 약 50만원 | 약 300만원 | 약 1,000만원 |

| 119 신고 앱 | 독립 앱 | 신고 기능 특화 | AED 검색 기능 미흡 |

### 8.2 심장지킴이의 차별점

| 차별점 | 설명 |
|--------|------|
| **토스 생태계 접근성** | 별도 앱 설치 없이 토스 앱 내에서 즉시 사용 가능. 3,000만 토스 사용자에게 노출돼요 |
| **모던 UX/UI** | TDS 기반 깔끔하고 직관적인 디자인. 기존 공공 서비스 대비 UX 월등히 우수해요 |
| **원탭 응급 대응** | 119 신고, AED 찾기, CPR 가이드를 한 화면에서 원탭으로 접근할 수 있어요 |
| **메트로놈 기능** | CPR 가슴 압박 리듬 가이드 (분당 100~120회)를 소리로 제공해요 |
| **오프라인 대응 (2차)** | 네트워크 불안정 시에도 최근 데이터로 AED 위치 확인이 가능해요 |
| **커뮤니티 데이터 보완 (2차)** | 사용자 제보로 공공 데이터의 오류를 실시간 보완할 수 있어요 |
| **공익 포지셔닝** | 순수 공익 서비스로 토스 플랫폼 내 노출/추천 우대를 기대할 수 있어요 |

### 8.3 핵심 성공 지표 (KPI)

| 평균 세션 시간 | 1분 30초 | 이벤트 로깅 |
