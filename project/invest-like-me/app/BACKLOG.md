# 콘솔 등록 후 해야 할 작업 백로그

## 1. 앱 기본 설정

### 1-1. CLIENT_ID 설정
- **파일**: `src/utils/auth.ts` (16번째 줄)
- **작업**: 콘솔에서 발급받은 `CLIENT_ID` 값을 `AUTH_CONFIG.CLIENT_ID`에 입력
- **현재**: `CLIENT_ID: ''` (빈 문자열)

### 1-2. 앱 아이콘 CDN URL 교체
- **파일**: `granite.config.ts` (8번째 줄)
- **작업**: 콘솔 앱 정보에서 업로드한 아이콘 이미지를 우클릭 → 링크 복사 → `icon` 값 교체
- **현재**: `icon: '/app-icon.svg'` (로컬 경로)

### 1-3. appName 확인
- **파일**: `granite.config.ts` (4번째 줄)
- **작업**: 콘솔에 등록한 앱 ID와 `appName` 값이 동일한지 확인
- **현재**: `appName: 'invest-like-me'`

---

## 2. 인증 및 서버

### 2-1. mTLS 인증서 기반 백엔드 구축
- **파일**: `src/utils/auth.ts`
- **작업**: 콘솔에서 mTLS 인증서(.pem) 다운로드 → 백엔드 `/api/auth/token` 엔드포인트 구현
- **설명**: 클라이언트 → 백엔드(mTLS) → auth.toss.im 토큰 교환 프록시

### 2-2. OAuth 콜백 테스트
- **파일**: `src/utils/auth.ts`, `src/contexts/AuthContext.tsx`
- **작업**: CLIENT_ID 설정 후 토스 로그인 → 콜백 → 토큰 발급 정상 동작 확인

---

## 3. 공유 기능

### 3-1. intoss:// 딥링크 주소 확인
- **파일**: `src/utils/share.ts` (145번째 줄)
- **작업**: `APP_LINK` 값이 실제 콘솔 등록된 앱의 딥링크와 일치하는지 확인
- **현재**: `intoss://miniapp?appId=invest-like-me`

### 3-2. 토스 네이티브 공유 API 활성화
- **파일**: `src/utils/share.ts` (148~153번째 줄)
- **작업**: OAuth 연동 완료 후 주석 해제하여 토스 네이티브 공유 1순위로 사용

---

## 4. 앱 내 기능 등록 (콘솔)

비게임 앱은 최소 1개 이상 등록 필수. 콘솔 → 앱 출시 → 앱 내 기능에서 등록.

| 기능 이름 | intoss 경로 |
|---|---|
| 투자 성향 테스트하기 | `intoss://{appName}/test` |
| 보조지표 도감 보기 | `intoss://{appName}/encyclopedia` |
| 학습 현황 확인하기 | `intoss://{appName}/progress` |

---

## 5. 콘솔 부가 설정

### 5-1. 고객센터 링크 등록
- **위치**: 콘솔 → 앱 정보
- **설명**: 네비게이션 바 더보기 메뉴의 "문의하기"에 자동 노출

### 5-2. 홈페이지 주소 등록
- **위치**: 콘솔 → 앱 정보
- **설명**: 네비게이션 바 더보기 메뉴에 자동 노출

## 6. 출시 전 최종 확인

### 6-1. 샌드박스 테스트
- 콘솔에서 테스터 등록 → 토스 앱 샌드박스 모드에서 전체 플로우 확인
- 네비게이션 바 정상 표시, 뒤로가기 이벤트, Storage 동작 확인

### 6-2. ait 번들 빌드 및 업로드
- `npm run build` → dist 폴더를 ait 번들로 패키징 → 콘솔에 업로드

### 6-3. 비게임 출시 체크리스트 최종 점검
- 네비게이션 바 중복 없는지 확인 (커스텀 뒤로가기 버튼 이미 제거됨)
- TDS 컴포넌트 사용 확인
- 면책 고지 위치 확인 (홈, 결과 페이지)
- 접근성 명도 대비 4.5:1 이상 확인
